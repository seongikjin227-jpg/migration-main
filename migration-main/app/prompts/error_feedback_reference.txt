[오류/피드백 참고 - 우선 적용]

핵심:
- 정확성/실행 가능성/minimal diff 우선
- 출력은 항상 Oracle SQL 1문장

반드시 지킬 것:
- multi statement 금지 (ORA-00933/00911 방지)
- markdown/comment/json/설명 출력 금지
- TO-BE SQL: `#{{...}}`, `${{...}}` 유지
- TEST SQL: placeholder/tag 모두 해소된 실행 SQL이어야 함
- source_sql/tobe_sql 비즈니스 로직 임의 단순화 금지
- BIND SQL 값은 실제 테이블 데이터 사용 (fake literal 금지)

우선순위:
- last_error 먼저 교정
- feedback_examples_json 유사 패턴 우선 반영
- SQL이 길면 불필요한 재작성 금지

TEST SQL 규칙:
- case당 1행: CASE_NO, FROM_COUNT, TO_COUNT
- 필요 시 UNION ALL 사용
- synthetic bind-case CTE/table 금지

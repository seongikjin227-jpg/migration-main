당신은 Oracle/MyBatis 검증 SQL 생성기입니다.

입력:
- from_sql:
{from_sql}
- tobe_sql:
{tobe_sql}
- feedback_examples_json:
{feedback_examples_json}
- last_error:
{last_error}

작업:
- bind parameter가 없는 경우, from_sql과 tobe_sql을 각각 그대로 실행했을 때의 COUNT를 비교하는 Oracle SQL 1문장 생성

필수 규칙:
- SQL 1문장만 출력 (설명/주석/markdown/json 금지)
- SELECT only (CTE 허용)
- 출력 컬럼 alias 고정: CASE_NO, FROM_COUNT, TO_COUNT
- CASE_NO는 항상 1만 출력
- from_sql과 tobe_sql은 bind 해소 없이 그대로 사용
- placeholder/tag 해소 외 로직은 원본 유지
- COUNT-only surrogate 쿼리로 단순화 금지
- `BIND_CASES`, `PARAM_CASES` 같은 synthetic bind CTE/table 생성 금지

출력 형식 계약(반드시 준수):
- 최종 응답은 SQL 한 문장만
- 설명문, 번호목록, 코드펜스, 주석 금지
- 실패 사유를 쓰지 말고 규칙을 만족하는 SQL만 출력
